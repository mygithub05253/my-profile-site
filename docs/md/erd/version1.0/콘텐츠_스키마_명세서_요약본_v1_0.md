## 핵심 규칙
- **slug**: PK, 파일명 일치, 전역 UNIQUE, 불변(+aliases), 한글·점 허용
- **상태 전이**: draft → ready → published → synced (역방향 금지, push 성공 후만 전환) / projects는 draft → published
- **노출 조건**: `visibility=public AND status IN (published, synced)`
- **projects 제약**: scope=team ⇒ role 필수 · private ⇒ GitHub 버튼 비노출 · velog-publish 경로 제외 · **status는 draft/published만(P-5) · slug 스캔에 projects 포함(P-6) · type⇔경로 일치(P-7)** (검토 F-1~F-3 반영)
- **빌드 순서 고정**: sync-content(clone) → velite build → next build
