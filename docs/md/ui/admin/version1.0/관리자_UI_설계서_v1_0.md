> **작성일:** 2026-07-04 (KST) | 상태: ■ **확정** (D-2·R-1·R-2·R-4 반영 완료, 2026-07-04 — 버전업은 구현 마일스톤 후 일괄)
> 대상: profile-admin 웹 (별도 레포 — ADR-9). 화면 A-01~A-05
> 디자인: **사이트와 동일 토큰 재사용 — npm 패키지 공유 확정** (R-4, ADR-10: `@mygithub05253/profile-tokens`, GitHub Packages). 관리 UI 특성상 spacing 1단계 축소 허용

## 1. 화면 ID 및 내비게이션

| 화면 ID | 라우트 | 화면명 | 우선순위 |
|---------|--------|--------|----------|
| A-01 | `/login` | 관리자 로그인 | P0 |
| A-02 | `/projects` | 프로젝트 관리 목록 | P0 |
| A-03 | `/projects/[slug]`·`/projects/new` | 프로젝트 편집/생성 | P0 |
| A-04 | `/drafts` | 블로그 초안 관리 | P1 |
| A-05 | `/dashboard` | 발행 대시보드 | P2 |

- 전역: 좌측 사이드바(Projects·Drafts·Dashboard) + 상단 사용자 메뉴(로그아웃). 미인증 시 전 라우트 → A-01

## 2. A-01 로그인

- 중앙 카드: 로고 + [GitHub로 로그인] 버튼 1개
- 에러 상태: OAuth 실패/취소 배너 · **allowlist 거부 = "접근 권한이 없습니다" 403 고정 문구** (사유 미노출 — FR-M16)

## 3. A-02 프로젝트 관리 목록

```
h1 Projects (N)                    [+ 새 프로젝트]
검색(제목) · 필터(status / category)
┌ 테이블: 제목 | category | scope | status | featured | 수정일 | ⋮ ┐
```
- 행 클릭 → A-03. ⋮: 편집/삭제
- 삭제 = 확인 모달 + **slug 재입력** (파괴 조작 2중 확인)
- D-2 확정(auto-merge PR): 행에 "병합 대기(CI 진행 중)" 상태 배지 표시

## 4. A-03 프로젝트 편집

```
┌ 좌: frontmatter 폼 ────────────┬ 우: MDX 에디터 ──┐
│ title · slug(신규만 편집 가능)  │ [작성|프리뷰] 탭  │
│ category(다중)·scope(단일)     │                  │
│ role(scope=team 시 필수 표시)  │                  │
│ period·stack(태그)·summary    │                  │
│ github·demo·repoVisibility·… │                  │
├────────────────────────────────┴─────────────────┤
│ 상태줄: idle→validating→committing→done/error [저장] │
```
- zod 인라인 필드 에러 (P-1·P-2·P-5 포함). dirty 이탈 경고
- **이미지 업로드 위젯 (FR-M22 확정)**: 에디터 하단 드롭존 — 드롭 시 클라이언트 리사이즈(최대 폭 1600px)·WebP 변환 → `assets/{slug}/` 경로 미리보기 + 본문 삽입 스니펫 복사. 장당 5MB 초과 시 즉시 거부 토스트. 저장 시 본문과 **단일 커밋**에 포함
- **409 충돌**: diff 패널 + [덮어쓰기/취소] (FR-M21 — 자동 재시도 3회 실패 후 노출)
- 502: 작성 내용 localStorage 임시 보존 + 재시도 안내
- 모바일: 폼/에디터 세로 스택 (데스크톱 우선, 붕괴 없음)

## 5. A-04 초안 관리 (P1)

- drafts 목록 + 편집(A-03 폼 재사용, posts 스키마)
- [발행 준비] → 확인 모달 "**velog에 자동 발행됩니다**" → drafts→posts 이동 커밋

## 6. A-05 발행 대시보드 (P2)

- 상단: 워크플로 3종 최근 실행 카드 (성공/실패·시각·로그 링크)
- 하단: 글별 상태 매트릭스 (status × velog_url × 사이트 노출)
- D-2 (b)안: admin발 PR CI/병합 상태 목록

## 7. 공통 UI 정책

- 로딩: 스피너 200ms 지연 노출 / 에러: 배너+재시도 / 성공: 토스트 + 커밋(PR) 링크
- 접근성: 사이트와 동일 기준 (AA·포커스·키보드) — 관리자 1인이어도 준수
- 반응형: 데스크톱 우선. 사이드바는 <1024에서 햄버거 수납
