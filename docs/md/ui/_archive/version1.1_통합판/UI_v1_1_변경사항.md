# UI 통합설계서 v1.1 변경사항 (v1.0 → v1.1)

> 2026-07-04 (KST) · Minor 버전업 (내용 승계 + 격식 승격 + admin 화면 신설)

## 버전업 판정 근거

v1.0의 4개 문서(요구사항 정의서·디자인시스템 검토서·DESIGN.md·UI 명세서)는 확정 내용이 유효하나, 화면 단위 정본이 분산되어 있었다. 캡스톤 설계서 5장(화면 설계) 격식의 **화면 ID 체계 통합본**으로 승격 — 내용 파기 없는 구조 개선이므로 Minor(+0.1).

## 변경 내역

| CN | 분류 | 내용 |
|----|------|------|
| CN-1 | 신규 | 화면 ID 체계(S-01~S-90, A-01~A-05) + 사이트맵·스토리보드 다이어그램 |
| CN-2 | 승격 | UI 명세서 v1.0의 와이어프레임을 화면별 설계(상태 정의 포함)로 확장 — S-02 상태 4종, S-03 버튼 조건부 렌더 규칙 명문화 |
| CN-3 | **신규** | 관리자 화면설계 A-01~A-05 — 로그인 403 정책, 편집 화면 2패널 레이아웃, 삭제 slug 재입력, 409 diff 패널, 초안 승격 경고 모달 |
| CN-4 | 신규 | 컴포넌트 인벤토리에 상태(state) 컬럼 추가, EmptyState·PrevNextNav 컴포넌트 추가 |
| CN-5 | 정책 확정 | 푸터 자동화 서사 문구를 자동화 생태계 프로젝트 상세 링크로 연결 / 햄버거 오버레이 포커스 트랩·스크롤 잠금 / 카드 내부 중첩 링크 금지 |
| CN-6 | 구조 | DESIGN.md는 토큰 정본으로 유지(재서술 금지), 본 문서는 참조만 — 이중 정의 방지 |

- `ui/version1.0/DESIGN.md`: **계속 유효** (토큰 정본, 구현 시 레포 루트 복사)
- `ui/version1.0/프로필사이트_재설계_요구사항_초안.md`: 계속 유효 (요구사항 정본)
- `ui/version1.0/UI_명세서.md`·`디자인시스템_검토서.md`: 본 v1.1이 승계 — 이력 참조용
