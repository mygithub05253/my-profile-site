# 프로필 사이트 디자인 시스템 검토서 v1.0

> 작성일: 2026-07-04 (KST) · 상태: **확정 — C안(Signal) 선택 (2026-07-04)**
> 포인트색은 사용자 위임으로 Signal Blue(코발트) 선정, 폰트 Pretendard Variable + JetBrains Mono
> 확정 토큰·규격: 같은 폴더 `DESIGN.md` (구현 시 레포 루트로 복사)
> 전제: 요구사항 정의서 v1.0 확정 (페르소나 A: 채용담당자 30~60초 스캔이 1순위)

## 1. 현행(v2) 진단

| 항목 | 현행 | 문제 |
|------|------|------|
| 색 | 다크 배경 + 시안 포인트 (이관 시 관성 유지) | 의도 없이 승계됨. 시안은 개발자 포트폴리오 최다 사용색 → 차별성 없음 |
| 타이포 | 시스템 폰트 위주 | 한글 본문/제목 위계 설계 없음 |
| 레이아웃 | 단일 컬럼 나열 | 섹션 리듬(밀도 변화·강조) 없음, 스캔 동선 미설계 |
| 모션 | 없음 | 첫인상 밋밋. 단, 과한 모션은 30초 스캔에 역효과 — 절제 필요 |
| 토큰 | 없음 (Tailwind 기본값) | 색·간격·타이포를 CSS 변수/토큰으로 관리 안 함 |

## 2. 브랜드 요구 정리

- **정체성**: 금융권 지향 풀스택 + AI·Data + "자동화 생태계를 직접 설계·운영하는 사람"
- **톤**: 신뢰(금융) ↔ 엔지니어링 감각(자동화) 균형. lova-clover식 캐릭터 친근함은 배제하되 경직되지 않게
- **제약**: 다크모드 우선(현행 사용자 인지 유지), Mobile First, Tailwind v4 토큰 체계와 호환

## 3. 방향 시안 3안

### A안 — "Terminal Trust" (다크 유지 + 색만 재설계) · 공수 小
- 배경 `#0B0E14`(청흑) / 본문 `#C9D1D9` / 포인트 **앰버-골드 `#E8B44F`**(금융 신뢰+차별성) / 보조 그린 `#3FB68B`(상승·성공 시맨틱)
- 타이포: 제목 Pretendard(가변) + 코드/수치 JetBrains Mono. 수치·성과는 모노스페이스로 "데이터 감성" 강조
- 모션: 스크롤 진입 fade-up 1회성(120ms stagger)만. 커서 그라디언트 등 장식 배제
- 인상: 블룸버그 터미널/쿼트 대시보드 — 금융+엔지니어 정체성에 가장 직결

### B안 — "Paper & Ink" (라이트 우선 전환) · 공수 中
- 웜 화이트 `#FAF8F5` + 잉크 블랙, 포인트 딥 네이비 `#1B3A5C`
- 세리프 제목(Noto Serif KR) + 산세리프 본문 — 리포트/백서 감성, 채용담당자 가독성 최상
- 리스크: 다크 우선 관성 폐기, OG 템플릿 등 재작업 범위 최대

### C안 — "Signal" (다크/라이트 듀얼 + 뉴트럴 베이스) · 공수 中上
- 뉴트럴 그레이 스케일 베이스 + 단일 포인트색(일렉트릭 바이올렛 `#7C5CFF`), 시스템 테마 자동 + 토글
- shadcn/ui 토큰 체계와 정합 최상, 확장성(관리자 웹 재사용) 최고
- 리스크: 구현 범위 가장 큼, 바이올렛은 금융 톤과는 거리

**권장: A안** — 정체성 부합·공수·현행 자산(다크 OG 템플릿) 재사용 균형이 가장 좋음. 관리자 웹에도 동일 토큰 재사용 가능.

## 4. 공통 토큰 초안 (방향 확정 후 값 대입)

```css
:root {
  --color-bg / -surface / -border / -text / -text-muted / -accent / -accent-soft / -success / -warning;
  --font-heading / -body / -mono;
  --text-xs~4xl (1.25 스케일) · --space-1~16 (4px 그리드) · --radius-sm/md/lg;
  --motion-fast: 150ms; --motion-base: 250ms; /* prefers-reduced-motion 존중 */
}
```

- 섹션 리듬: Hero(고밀도 임팩트) → About(저밀도 여백) → Projects(카드 그리드) 교차로 스캔 피로 감소
- 프로젝트 카드: 썸네일(next/og 재사용) + 카테고리 칩 + 스택 아이콘 3개 제한 + 한 줄 요약
- 접근성: 명도 대비 WCAG AA, 포커스 링 유지

## 5. 사용자 선택 대기

1. **방향 선택**: A(Terminal Trust, 권장) / B(Paper & Ink) / C(Signal) — 또는 A+C 절충(다크 우선 + 토글)
2. A안 선택 시 포인트색 확정: 앰버-골드 vs 현행 시안 유지(색상만 정제)
3. 제목 폰트: Pretendard(무난·가독) vs 개성 폰트 검토(예: SUIT, Wanted Sans)

> 선택 후: 토큰 값 확정 → 폰트/컬러 프리뷰 페이지 생성 → DESIGN.md(소스 오브 트루스) 작성 → UI 명세·와이어프레임 진행
