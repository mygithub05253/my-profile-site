### 6) Star(대표) 프로젝트 (V01, V02)

- `ProjectCard.tsx`: `project.featured === true`일 때 카드 우상단에 ⭐ 배지(절대 위치, 썸네일 위 오버레이) 추가. 카테고리·scope·비공개 칩은 기존처럼 콘텐츠 영역에 남아 있어 배지와 겹치지 않음.
- `app/page.tsx`: `featuredProjects` 선정 로직을 `getPublishedProjects().slice(0, 6)` → `getPublishedProjects().filter((p) => p.featured)`로 변경.
  - **엣지 케이스 (0건)**: Star가 0건이면 섹션이 비므로, 0건일 때만 기존 방식(`getPublishedProjects().slice(0, 6)`)으로 폴백.
  - **엣지 케이스 (과다)**: Star가 6건을 초과해도 홈은 하이라이트 목적이므로 `slice(0, 6)`으로 상한(초과분은 "모든 프로젝트 보기"에서 확인). 상한 적용 순서는 `getPublishedProjects()`의 기존 정렬(featured desc → order asc) 기준.
- `lib/projects.ts`: `ProjectSort` 타입(`"recommended" | "latest" | "title"`) + `sortProjects(list, sort)` 함수 신설.
  - `recommended`(기본): 기존 `getPublishedProjects()` 정렬(featured desc → order asc) 그대로.
  - `latest`: `period` 문자열에서 선행 `YYYY-MM`을 정규식(`/^\d{4}-\d{2}/`)으로 추출해 내림차순 비교. 추출 실패(빈 문자열·형식 불일치) 시 해당 항목은 비교값을 `""`(가장 낮은 우선순위)로 취급해 예외 없이 정렬 끝쪽으로 안정 정렬(`Array.prototype.sort`는 안정 정렬 보장).
  - `title`: `title.localeCompare(other.title, "ko", { sensitivity: "base" })` — 대소문자·자모 결합 차이 무시.
- `components/ProjectsExplorer.tsx`: 기존 `?filter=` 패턴과 동일하게 `?sort=` 쿼리 추가. 기존 `isFilter` 가드와 동일하게 `isSort(v): v is ProjectSort` 가드를 만들어 **알 수 없는/누락된 `sort` 값은 무조건 `"recommended"`로 폴백** — 잘못된 URL을 직접 입력해도 throw 없이 기본 정렬로 렌더. 필터 버튼 그룹 옆에 정렬 드롭다운(또는 세그먼트 버튼) 배치. `filterProjects` 결과에 `sortProjects`를 이어서 적용(필터 → 정렬 순서 고정, 필터링 후 목록에 대해서만 정렬).

### 7) 프로젝트 상세 화면 재설계 (V03)

- `velite.config.ts` projects 스키마에 optional 필드 추가:
  - `images: s.array(s.string()).optional()` — 썸네일 외 추가 스크린샷 경로 배열. `thumbnail`과 동일하게 `withLeadingSlash` 정규화 적용(배열 각 원소에 map).
  - `problem: s.string().optional()`, `build: s.array(s.string()).optional()`(불릿 목록), `result: s.array(s.string()).optional()`(불릿 목록) — 캡처 화면의 PROBLEM/BUILD/RESULT + "핵심 구현"/"성과와 배운 점" 카드에 대응.
- `lib/projects.ts`(또는 신규 `lib/project-sections.ts`)에 `getStructuredSections(project)` 헬퍼 추가:
  - `problem`/`build`/`result` 중 하나라도 frontmatter에 있으면 그 값 우선 사용. `build`/`result`는 배열 원소 중 공백(trim 결과 빈 문자열)만 있는 항목을 필터링하고, 필터링 후 빈 배열이면 해당 필드는 "없음"으로 취급.
  - 전부 비어 있으면 `project.html`(렌더된 HTML 문자열)에서 `<h2>문제 정의</h2>`, `<h2>구현</h2>`, `<h2>성과</h2>`, `<h2>트러블슈팅</h2>`, `<h2>배운 점</h2>` 5개 섹션을 파싱(정규식 또는 간단 DOM 파싱)해 `problem = 문제 정의`, `build = 구현`, `result = 성과 + 트러블슈팅 + 배운 점` 병합으로 매핑.
  - 파싱 실패(소제목 형식이 다른 과거/예외 콘텐츠, 5개 중 일부만 매칭되는 경우 포함) 시 `null` 반환 → 카드 섹션 자체를 렌더하지 않고 기존처럼 `project.html` 전체만 표시(현행 동작 유지, 회귀 없음). **부분 매칭은 "일부만 채우기"를 시도하지 않고 전부-아니면-폴백으로 단순화** — 어중간한 카드 레이아웃(빈 카드 노출)을 방지.
- `velite.config.ts`에서 `images` 배열도 `thumbnail`과 동일하게 `withLeadingSlash` 정규화 + **공백/빈 문자열 원소 제거** 후 저장(원본 mdx에 실수로 빈 줄이 들어가도 방어).
- `app/projects/[slug]/page.tsx` 레이아웃 변경:
  - 헤더(제목·카테고리·기간) 아래에 이미지 영역: `project.images`가 2장 이상이면 캐러셀(클라이언트 컴포넌트, 이전/다음 버튼 + 점 인디케이터), **정확히 1장이면 이전/다음 버튼 없이 정적 이미지 1장만**(불필요한 죽은 컨트롤 방지), 없고 `thumbnail`만 있으면 단일 히어로 이미지, 둘 다 없으면 이미지 영역 자체 미노출.
  - 이미지 `<img>`에는 `onError`로 로드 실패 시 `ProjectCard`와 동일한 파스텔 플레이스홀더로 교체(경로 오타·404로 깨진 이미지 아이콘이 그대로 노출되는 것을 방지 — 기존 설계에서 "범위 밖"으로 뒀던 부분을 상향).
  - 그 아래 PROBLEM / BUILD / RESULT 3카드 — `getStructuredSections` 결과가 있을 때만 렌더.
  - 그 아래 기존 `project.html` 전체 본문은 그대로 유지(요약 카드가 있어도 원문 상세는 스크롤해서 볼 수 있게 — README 상세 내용 유실 방지, 사용자 피드백 반영).
- **접근성(V07 기존 기준 준수)**: 캐러셀은 좌우 화살표 키보드 내비게이션 지원, 현재 슬라이드 위치를 `aria-live="polite"`로 스크린리더에 안내, 점 인디케이터 각각에 `aria-label="n번째 이미지로 이동"`. 정렬 드롭다운·이전/다음 버튼도 `aria-label` 필수(기존 `ThemeToggle`·`ProjectsExplorer` 필터 버튼과 동일한 접근성 수준 유지).
- content-hub 쪽 계약(문서화만, 이번 라운드 미구현): `projects/*.mdx` frontmatter에 `images: string[]`, `problem: string`, `build: string[]`, `result: string[]` optional 필드 추가 가능. 실제 11개 파일 반영은 콘텐츠 작업(사용자 또는 별도 세션)에서 진행.
