## 1.1 프로젝트 소개

### ■ 프로젝트 정의

본 프로젝트는 **"한 번 쓰면 모든 곳에 반영된다"**를 목표로 하는 개인 브랜딩 자동화 생태계와, 그 위에서 동작하는 포트폴리오 프로필 사이트를 설계·구현·운영하는 프로젝트이다.

- **자동화 생태계**: Obsidian에서 작성한 글이 content-hub(SSOT)로 동기화되고, velog 자동 발행 → Vercel 사이트 재빌드 → GitHub 프로필 README 갱신까지 사람의 개입 없이 전파되는 CI/CD 파이프라인. **2026-07-04 전 구간 E2E 검증 완료 (100%)**
- **프로필 사이트 재설계**: Phase 2의 "설계 없는 이관"을 폐기하고, 요구사항 → 프로젝트 선정(11건) → IA/유스케이스 → 콘텐츠 스키마 → 디자인 시스템(C안 Signal) → UI 명세 → 구현의 정식 프로세스로 재구축
- **관리자 웹(profile-admin)**: content-hub의 프로젝트/글 MDX를 웹 UI로 CRUD하는 서버리스 관리자 도구 (신규 기획, 그 자체가 포트폴리오 프로젝트)

### ■ 프로젝트 목표

첫째, 콘텐츠 발행 파이프라인을 완전 자동화하여 글쓰기 이외의 반복 작업(발행·배포·프로필 갱신)을 0으로 만든다. — **달성 (E2E 완료)**
둘째, 채용담당자가 30~60초 내에 기술 스택·핵심 프로젝트·성과를 판단할 수 있는 프로필 사이트를 구축한다.
셋째, 프로젝트 11건을 사이트 내 정적 상세 페이지(STAR 서사)로 게재하고, 6종 필터로 탐색 가능하게 한다.
넷째, 코드 수정 없이 콘텐츠를 관리하는 관리자 웹을 설계·구현한다 (DB 없는 GitHub-API 기반 서버리스).

### ■ 프로젝트 진행 방법

1인 개발 + Claude Code 협업 체계로 진행하며, 다음 규칙을 따른다.

- **Git 흐름**: main 직접 커밋 금지. 브랜치 → PR → 사용자 승인 병합
- **문서 흐름**: 작업 중 변경 노트 누적 → 기능/마일스톤 완료 시 일괄 버전업(통합본+요약본+변경사항 3종 세트)
- **검증 흐름**: 구현 → 빌드 검증 → 로컬 프리뷰 → E2E → /qa · /design-review

## 1.2 배경 및 문제의식

1. **콘텐츠 분산 문제**: velog·GitHub·Obsidian·프로필 사이트에 콘텐츠가 분산되어 이중 작성과 누락이 발생 → content-hub 단일 소스(SSOT)로 통합
2. **기존 사이트의 한계**: 정적 HTML → Next.js 이관(Phase 2)은 설계·프로젝트 선정·상세 페이지·디자인 시스템이 부재한 "겉모습 이전"에 불과 (사용자 피드백)
3. **포트폴리오 서사 부재**: 62개 레포 중 무엇을 어떻게 보여줄지 기준이 없었음 → Tier 매트릭스 분석으로 11건 확정

## 1.3 기대효과

첫째, **운영 비용 최소화** — 발행·배포·README 갱신 자동화로 콘텐츠 생산에만 집중.
둘째, **채용 관점 전달력** — 30초 스캔 IA + STAR 상세 페이지 + 성과 수치 강조(mono+success 토큰).
셋째, **자동화 생태계 자체가 포트폴리오** — 사이트 푸터·프로젝트 상세에 파이프라인 서사 노출.
넷째, **확장성** — 관리자 웹, 시리즈/태그 탐색, 분석 대시보드 등으로 확장 가능한 토큰·스키마 기반 설계.

## 1.4 개발 목표와 범위

### ■ 개발 범위

**(1) 자동화 파이프라인 (완료, as-built)**
- Obsidian `70_Blog/` → sync-blog.ps1(작업 스케줄러 09:00/21:00) → content-hub push
- content-hub: frontmatter 검증 CI + velog 자동 발행(GraphQL) + auto-sync 브랜치 + deploy-site(Vercel Hook + README dispatch)
- velog-backup: 일일 GraphQL 백업 + 주간 역방향 PR
- mygithub05253: BLOG-POST-LIST 마커 README 자동 갱신 (dispatch + 일일 스케줄)

**(2) 프로필 사이트 재설계 (본 설계서의 구현 대상)**
- 디자인 토큰(C안 Signal, 듀얼 테마) 적용 + 테마 토글
- 홈 재구성: Hero → 강점 4카드 → Stack → Featured Projects → Records 타임라인 → Blog 3건 → Contact
- `/projects` 목록(필터 6종, URL 쿼리 동기화) + `/projects/[slug]` 정적 상세(글별 OG)
- content-hub `projects/` 스키마 신설 + Velite projects 컬렉션 + velog-publish 경로 필터

**(3) 관리자 웹 profile-admin (설계 → 구현 후순위)**
- GitHub OAuth 1인 인증, 프로젝트 CRUD(P0) → 블로그 초안 관리(P1) → 발행 대시보드(P2)
- 저장은 GitHub Contents API로 content-hub 커밋 (DB 없음, 기존 파이프라인이 재빌드 담당)

### ■ 범위 제외 (Out of Scope)

- Vault 전체 git화 (26,000+ 파일, Google Drive 동기화 폴더 — 금지)
- contentlayer / next-mdx-remote 사용 (Velite 확정)
- 다국어(i18n), 댓글 시스템, 방문자 분석 백엔드(현 단계)
