# features/ — 기능명세서

## 이 폴더에 있는 것

프로젝트의 **요구기능 정의(FR)와 비기능 요구사항(NFR)** 정본. 화면·API·스키마 문서가 "어떻게"를 다룬다면, 여기는 "무엇을 왜"를 다룬다.

| 경로 | 내용 |
|------|------|
| `version2.0/user/` | 사용자(방문자) 기능명세 — FR-V 8종 (현행 정본) |
| `version2.0/admin/` | 관리자 기능명세 — 자동화 운영 FR-A 7종 + profile-admin FR-M 8종 (현행 정본) |
| `version1.1/` | 자동화 생태계 아키텍처 설계서 — **동결** (architecture/erd/api 도메인으로 승계) |
| `version1.0/` | 초기 설계서 + Perplexity 리서치 프롬프트 — 이력 |
| `_archive/version2.0_통합판/` | 사용자/관리자 분리(2026-07-04) 전 통합판 — 이력 |

## 버전 관리 방식

- **`versionX.Y/` 폴더 단위 버전 관리**: Major(+1.0) = 구조 개편·도메인 재정의, Minor(+0.1) = 기능 추가·정정
- 각 버전은 **3종 세트**(통합본 · 요약본 · 변경사항)를 동시 유지 — 통합본이 정본, 요약본은 빠른 참조, 변경사항은 이전 버전 대비 diff
- **작업 중에는 버전업하지 않는다**: 스펙 변경은 기존 버전 문서에 갱신 + 변경사항 파일에 CN 번호로 누적 → 기능·마일스톤 완료 후 한 번에 버전업
- 구버전은 삭제하지 않고 `_archive/` 또는 동결 표기로 보존
- 최신 버전 현황은 [`../프로젝트_문서_마스터인덱스.md`](../프로젝트_문서_마스터인덱스.md) 스냅샷 표가 단일 기준
