# Perplexity 리서치 프롬프트 패키지 v1.0

> 용도: Phase 0 시장/기술 조사. 각 프롬프트는 독립 실행 가능.
> 공통 권장: **"자동화 생태계 구축"이라는 Perplexity 스페이스를 새로 생성**하고 아래 리서치를 모두 그 스페이스 안에서 진행 (컨텍스트 누적 활용).

---

## 리서치 1: velog 비공식 API 2026년 현황 (최우선 — 아키텍처 분기점)

- **세션**: 스페이스 내 **새 스레드** 생성 (독립 주제)
- **모델**: 기본 모델 + **심층 리서치(Deep Research) 모드 ON** — 비공식 API는 최신 커뮤니티 자료 수집이 관건
- **첨부**: 불필요 (Computer/파일 도구 미사용)
- **프롬프트**:

```
velog(velog.io)의 비공식 GraphQL API를 이용한 글 자동 발행에 대해 2025~2026년 최신 현황을 심층 조사해줘.

1. 현재 동작하는 GraphQL 엔드포인트 주소와 writePost/editPost 뮤테이션 스펙 (v3.velog.io/graphql 유효 여부)
2. 인증 방식: access_token/refresh_token 쿠키 획득 방법과 토큰 유효기간, GitHub Actions 등 헤드리스 환경에서의 토큰 갱신 전략
3. 2025년 이후 API 스펙 변경/차단 사례와 커뮤니티(GitHub 오픈소스, 블로그)의 대응 방법
4. velog 자동 발행을 구현한 최신 오픈소스 레포 3~5개와 각각의 유지보수 상태(마지막 커밋, 이슈)
5. 자동 발행이 velog 약관에 저촉될 가능성과 실제 제재 사례 유무

결과는 "완전 자동 발행이 현재 안정적으로 가능한가 (예/아니오/조건부)"라는 결론과 근거로 정리해줘.
```

## 리서치 2: Notion/Obsidian → GitHub → 블로그 파이프라인 사례 벤치마킹

- **세션**: 같은 스페이스, **새 스레드**
- **모델**: 기본 모델 + **심층 리서치 모드 ON**
- **첨부**: 불필요
- **프롬프트**:

```
개발자들이 Notion 또는 Obsidian을 글쓰기 도구로 쓰고, GitHub Actions로 개인 블로그(velog, GitHub Pages, 개인 사이트 등)에 자동 발행하는 파이프라인의 실제 구축 사례를 심층 조사해줘.

1. Notion API 기반 파이프라인의 대표 오픈소스/블로그 사례 (블록 파싱, 이미지 URL 만료 처리 방법 포함)
2. Obsidian 기반: obsidian-git 플러그인으로 Vault를 GitHub 레포와 동기화하고 이를 발행 소스로 쓰는 사례와 주의점 (모바일 동기화, 충돌 처리)
3. Notion과 Obsidian을 동시에 소스로 쓰는 경우의 충돌/중복 처리 패턴
4. frontmatter 스키마 설계 모범 사례 (slug, 시리즈, 발행 상태 관리)
5. 실패했거나 유지보수를 포기한 사례에서의 교훈

각 사례의 아키텍처 다이어그램 수준 요약과 GitHub 링크를 포함해줘.
```

## 리서치 3: 개발자 포트폴리오 사이트 트렌드 (Next.js 15 기준)

- **세션**: 같은 스페이스, **새 스레드**
- **모델**: 기본 모델, 심층 리서치 **불필요** (일반 검색으로 충분, 빠른 응답 우선)
- **첨부(Computer/파일 도구 활용 시)**: 현재 사이트 스크린샷(`index.html` 렌더 캡처) + `docs/md/features/version1.0/자동화_생태계_아키텍처_설계서.md`의 6장 부분을 첨부하면 비교 분석 품질 상승
- **프롬프트**:

```
2025~2026년 개발자 개인 포트폴리오 사이트의 트렌드를 조사해줘. 나는 Next.js 15(App Router) + Tailwind + shadcn/ui로 재구축 예정이고, 블로그(velog 연동)·회고·프로젝트 소개 섹션을 넣을 계획이야.

1. 신입~주니어 개발자 포트폴리오에서 채용담당자에게 효과적이라고 평가받는 섹션 구성과 서사 구조
2. Next.js 포트폴리오의 콘텐츠 관리 트렌드: contentlayer 대체재(contentlayer2, velite, next-mdx-remote 등) 비교와 2026년 권장안
3. 정적 export(GitHub Pages) vs Vercel 배포의 실질적 차이 (개인 포트폴리오 기준 비용/기능)
4. 참고할 만한 한국 개발자 포트폴리오 우수 사례 5개 (velog/GitHub 연동형 우선)

첨부한 내 현재 사이트/설계서와 비교해 개선 포인트를 제안해줘.
```
