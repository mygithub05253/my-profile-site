## 11. Phase 1~3 구현 결과 (as-built, v1.1 신규)

### 11.1 Phase 1 — 콘텐츠 파이프라인 (2026-07-03 완료)
- content-hub 생성(Public): frontmatter 검증 + velog 발행 모듈 + CI. velog 실인증·실발행 E2E 성공
- velog 백필 48건 수집 → main 총 49편. 5분 10개 초과 자동 비공개 대응 스로틀 적용
- velog-backup 주간 역방향 PR 워크플로. Obsidian `70_Blog` + sync-blog.ps1 스케줄러 등록

### 11.2 Phase 2 — 사이트 재구축 (2026-07-03 완료)
- Next.js 15 App Router + TS Strict + Tailwind v4 + Velite. 기존 정적 HTML 삭제
- 빌드 54페이지 SSG. 섹션 구성 10.3 준수. Vercel 배포: https://my-profile-site-coral.vercel.app
- content-hub deploy-site.yml → Vercel Deploy Hook 배선 E2E 검증 → **핵심 루프(발행→재빌드) 완성**

### 11.3 Phase 3 — 썸네일 + README (2026-07-04 E2E 완료분)
- **OG 썸네일 (my-profile-site PR #2 병합)**: next/og 글별+기본. 프로덕션 검증 —
  홈 `/opengraph-image` 200 image/png(1200×630), 글별 `/blog/{slug}/opengraph-image` 200 image/png
- **README 갱신 배선 (mygithub05253 PR #1, content-hub PR #7 병합)**: BLOG-POST-LIST 마커 추가(기존 무동작 버그 수정) + dispatch 수신/송신 배선
- **E2E 중 발견 결함 2건**:
  1. `CONTENT_DISPATCH_TOKEN`이 mygithub05253 레포 스코프 없음 → dispatch 403 (notify-profile-readme 잡 실패). trigger-vercel 잡은 성공 — 잡 분리 덕에 장애 격리(NFR-03) 확인
  2. blog-post.yml의 기본 GITHUB_TOKEN이 read-only → README 커밋 push 403 (스케줄 실행도 수일째 실패 중이었음). RSS 수집·마커 치환은 정상(5줄 삽입 확인) → `permissions: contents: write` 추가 PR #2 생성
- dispatch 수신 검증: 로컬 gh 인증으로 dispatch 직접 전송 → blog-post.yml repository_dispatch 트리거 확인 (수신 배선 정상)

### 11.4 운영 교훈 (재발 방지)
- 워크플로가 레포에 쓰기(push)하면 **반드시 `permissions: contents: write` 명시** (기본 토큰 read-only)
- fine-grained PAT는 **대상 레포별로 명시 등록** — dispatch 대상 추가 시 PAT 스코프도 갱신
- PS 5.1에서 `gh pr create` 본문은 `--body-file` 사용 (here-string 인수 깨짐)
- 원격에 `auto-sync` 브랜치 존재 시 `auto-sync/xxx` 계층 브랜치 생성 불가 → 평면명 사용
