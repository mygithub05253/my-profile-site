# public/

Next.js가 그대로 정적 서빙하는 파일들. 이 레포에 커밋된 원본 이미지는 빌드와 무관하게 항상 존재해야 하는 것들만 둔다 — 프로젝트 썸네일처럼 content-hub에서 관리되는 이미지는 여기 없고, 빌드 시 `scripts/sync-content.mjs`가 `public/assets/`로 복사해 넣는다(이 폴더는 `.gitignore` 대상이라 레포에는 안 보인다).

## 구성

- `profile-bear.png` / `profile-bear.svg` — 홈 Hero의 곰돌이 아바타(GPT Image 생성본). 사이트 브랜드 마스코트라 콘텐츠 교체 주기와 무관하게 고정 자산으로 취급해 이 레포에 직접 커밋했다.

## 정리 원칙

"콘텐츠는 content-hub에만 둔다" 원칙의 예외가 이 폴더다. 예외를 두는 기준은 명확하다 — 글이나 프로젝트처럼 자주 늘어나거나 바뀌는 콘텐츠가 아니라, 사이트 정체성 자체에 속하는 소수의 정적 이미지만 여기 둔다.
